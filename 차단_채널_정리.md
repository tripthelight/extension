# 차단 채널 경로 정리 및 필요 Data 배정

## 메인화면
메인화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-browse > ytd-two-column-browse-results-renderer > ytd-rich-grid-renderer`

### long
1. 메인화면에서 long 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 long 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
2. long tag flow : `메인화면 tag flow > ytd-rich-item-renderer`
   * long의 채널명 : <br>
    `long tag flow > yt-lockup-metadata-view-model > yt-content-metadata-view-model > a` 의 textContent(내가 찾는 text 바로 다음에 다른 text를 가진 다른 tag 있을 수 있음)
   * long의 채널 주소 : <br>
    `long tag flow > a` 중 href 문자열이 "/@" 로 시작하는 a의 href 문자열에서 시작문자열인 "/@" 를 제거한 후 나머지 문자열을 decodeURIComponent 한 결과 문자열
   * long 썸네일 클릭 시 이동하는 url(video-id) : <br>
    `long tag flow > a` 중 href가 "/watch" 로 시작하고, "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 long 영상의 video-id가 됨

### Shorts
1. 메인화면에서 Shorts 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 Shorts 영상의 \[ code, label, pageType, video-id \] 가 필요함
2. Shorts tag flow : `메인화면 tag flow > ytd-rich-section-renderer > ytd-rich-shelf-renderer > ytd-rich-item-renderer`
   * Shorts 썸네일 클릭 시 이동하는 url(video-id) : <br>
      `Shorts tag flow > a` tag 중 href가 "/shorts/" 로 시작하는 첫번째 a tag 의 href 문자열에서, 시작 문자열인 "/shorts/" 문자열를 제거한 후, 나머지 문자열이 해당 Shorts 영상의 video-id가 됨


## 검색 후 화면
검색 후 화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-search > ytd-two-column-search-results-renderer > ytd-section-list-renderer > ytd-item-section-renderer`

### long
1. 검색 후 화면에서 long 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 long 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
2. search long tag flow : `검색 후 화면 tag flow > ytd-video-renderer`
   * long의 채널명 : <br>
      `search long tag flow > ytd-channel-name > a` 중 href가 "/@" 로 시작하는 첫번째 a tag의 textContent(내가 찾는 text 바로 다음에 다른 text를 가진 다른 tag 있을 수 있음)
   * long의 채널 주소 : <br>
      `search long tag flow > ytd-channel-name > a` 중 href가 "/@" 로 시작하는 첫번째 a tag의 href 문자열에서 시작문자인 "/@" 문자를 제거하고 남은 문자열
   * long 썸네일 클릭 시 이동하는 url(video-id) : <br>
      `search long tag flow > a` 중 href가 "/watch" 로 시작하고 "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 video-id가 됨

### Shorts
1. 검색 후 화면에서 Shorts 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 Shorts 영상의 \[ code, label, pageType, video-id \] 가 필요함
2. search Shorts tag flow : `검색 후 화면 tag flow > grid-shelf-view-model > div.ytGridShelfViewModelGridShelfRow > div.ytGridShelfViewModelGridShelfItem`
   * Shorts 썸네일 클릭 시 이동하는 url(video-id) : <br>
      `search Shorts tag flow > a` tag 중 href가 "/shorts/" 로 시작하는 첫번째 a tag 의 href 문자열에서, 시작 문자열인 "/shorts/" 문자열를 제거한 후, 나머지 문자열이 해당 Shorts 영상의 video-id가 됨


## 상세화면
상세화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-watch-flexy`

### 해당영상
1. 상세화면에서 해당영상 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 해당영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
2. local tag flow : `상세화면 tag flow > ytd-watch-metadata > ytd-menu-renderer`
   * 해당영상의 채널명 : <br>
    `상세화면 tag flow > ytd-watch-metadata > ytd-video-owner-renderer > ytd-channel-name > yt-formatted-string > a` 의 textContent(내가 찾는 text 바로 다음에 다른 text를 가진 다른 tag 있을 수 있음)
   * 해당영상의 채널 주소 : <br>
    `상세화면 tag flow > ytd-watch-metadata > ytd-video-owner-renderer > ytd-channel-name > yt-formatted-string > a` 의 href 가 "/@" 로 시작한다면 "/@" 제거 후 나머지 문자열을 decodeURIComponent 한 결과 문자열
   * 해당영상 url(video-id) : <br>
    `상세화면 tag flow > ytd-watch-metadata` tag에 정의(attribute)된 의 "video-id" key 의 값 문자열

### 추천영상 long
1. 상세화면에서 해당영상 오른쪽에 추천된 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 해당영상의 \[ code, label, pageType, 채널명, video-id \] 가 필요함
2. recommend tag flow : `상세화면 tag flow > ytd-watch-next-secondary-results-renderer > ytd-item-section-renderer > yt-lockup-view-model`
   * 추천영상의 채널명 : <br>
    `recommend tag flow > yt-content-metadata-view-model > div:first-of-type > span` 의 textContent(내가 찾는 text 바로 다음에 다른 text를 가진 다른 tag 있을 수 있음)
   * 추천영상 클릭 시 이동하는 url(video-id) : <br>
    `recommend tag flow > a` 중 href가 "/watch" 로 시작하고 "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 video-id가 됨

### 재생목록
재생목록 tag flow : `상세화면 tag flow > ytd-playlist-panel-renderer`

#### 재생목록 상단 타이틀
1. 상세화면에서 해당영상 오른쪽에 표시되는 재생목록 상단 타이틀 영역에 있는 점3개 버튼 - 이 경우는 추천안함, 관심없음 버튼 필요없음
2. playlist header tag flow : `재생목록 tag flow > div#header`

#### 재생목록 리스트
1. 상세화면에서 해당영상 오른쪽에 표시되는 재생목록 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 해당영상의 \[ code, label, pageType, 채널명, video-id \] 가 필요함
2. playlist tag flow : `재생목록 tag flow > div#items > ytd-playlist-panel-video-renderer`
   * playlist 내부에서 선택한 채널명 : <br>
    `playlist tag flow > span#byline.style-scope.ytd-playlist-panel-video-renderer` 의 textContent(내가 찾는 text 바로 다음에 다른 text를 가진 다른 tag 있을 수 있음)
   * playlist 내부에서 선택한 영상 클릭 시 이동하는 url(video-id) : <br>
    `playlist tag flow > a` 중 href가 "/watch" 로 시작하고 "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 video-id가 됨


## Shorts 화면
Shorts 화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-shorts`

1. Shorts 화면에서 해당 Shorts 영상 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 Shorts 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
2. Shorts page tag flow : `Shorts 화면 tag flow > ytd-reel-video-renderer > ytd-player`
   * 해당 Shorts 채널명 : <br>
      * `Shorts page tag flow > yt-reel-channel-bar-view-model > .yt-core-attributed-string > a` 의 href 문자열은 "/@%EC%9B%90%EC%98%81%EC%9D%B4%EC%97%90_%EB%AF%B8%EC%B9%9C_bird/shorts" 형식으로 되어있음
      * "/@%EC%9B%90%EC%98%81%EC%9D%B4%EC%97%90_%EB%AF%B8%EC%B9%9C_bird/shorts" 는 설명을 위한 예시 문자열임.
      * href에서 시작 문자인 "/@"을 삭제하고, 끝 문자열인 "/shorts"를 삭제하고 남은 문자열을 decodeURIComponent 한 결과 문자열이 채널명이 됨
   * 해당 Shorts 채널 주소 : <br>
      * **해당 Shorts 채널명** 과 해당 Shorts 채널 주소는 동일함
   * 해당 Shorts url(video-id) : <br>
      * `Shorts page tag flow > .ytp-chrome-top > .ytp-title > .ytp-title-text > a.ytp-title-link` 의 href문자열은 "https://www.youtube.com/shorts/Pl7lB8X9scg" 형식으로 되어있음. 
      * "Pl7lB8X9scg" 는 설명을 위한 예시 문자열임. 
      * location.pathname 의 "/shorts/Pl7lB8X9scg" 와 href 문자열의 "/shorts/Pl7lB8X9scg" 부분이 같다면 "Pl7lB8X9scg" 문자열이 video-id가 됨
