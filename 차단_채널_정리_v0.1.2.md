# 차단 채널 경로 정리 및 필요 Data 배정

## 이 module의 목적 :
```javascript
{
   "채널명": "", // 점 3개 버튼을 클릭한 영상 썸네일의 채널명
   "채널 주소": "", // 점 3개 버튼을 클릭한 영상 썸네일의 채널 주소
   "channel-id": "", // 점 3개 버튼을 클릭한 영상 썸네일의 다른 채널 주소
   "video-id": "", // 점 3개 버튼을 클릭한 영상 썸네일을 클릭할 경우 이동하는 url
   "list-ref": "" // 점 3개 버튼을 클릭한 영상 썸네일의 리스트 html element tag 명
}
```
1. 유튜브 영상에있는 점 3개 버튼을 클릭했을 때, 클릭한 점 3개 버튼을 포함하고 있는 영상의 \[ 채널명 or channel-id, 채널 주소, video-id \] Data를 알아내야 한다.
```javascript
{
   "메인화면": [
      "long", // 채널명, 채널 주소 or channel-id, video-id, list-ref
      "Shorts" // video-id, list-ref
   ],
   "검색 후 화면": [
      "long", // 채널명, 채널 주소 or channel-id, video-id, list-ref
      "Shorts" // video-id, list-ref
   ],
   "상세화면": [
      "해당영상", // 채널명, 채널 주소 or channel-id, video-id
      "추천영상 long", // 채널명, video-id, list-ref
      "재생목록" // video-id, list-ref
   ],
   "Shorts 화면" : [
      "현재 재생중인 Shorts 영상" // 채널명, 채널 주소 or channel-id, video-id
   ],
   "채널페이지": {
      "홈 탭": ["최상단에서 현재 재생중인 영상", "실시간 스트림 n개", "그 하위영상 리스트", "Shorts"],
      "동영상 탭": ["채널페이지 동영상 탭 화면에서 영상 썸네일들 중 하나"],
      "Shorts 탭": ["채널페이지 Shorts 탭 화면에서 영상 썸네일들 중 하나"],
      "라이브 탭": ["채널페이지 라이브 탭 화면에서 영상 썸네일들 중 하나"]
   },
   "재생목록 팝업": [ // 상세화면의 재생목록 이었던 것
      "팝업형태로 떠있는 재생목록 리스트 중 하나의 영상 썸네일" // video-id, list-ref
   ]
}
```
2. Data가 필요한 영상들은 위와 같다.
   * 여기서 마지막에 있는 "재생목록 팝업"은 메인화면, 검색 후 화면, 상세화면, Shorts 화면, 채널페이지 모두의 위에 떠있는 playlist 팝업이다.
     - 이 팝업은 이전에 검색해서 상세로 진입했던 영상이 어떤 재생목록에 포함된 영상일 경우, 다른 화면으로 이동해도, 이동한 화면의 오른쪽 하단에 playlist 형태로 영상을 멈추지 않고 항상 떠있다.

3. 전략
   1) 점 3개 버튼을 클릭할 경우, 클릭한 점 3개 버튼에 해당하는 html element tag 를 포함하고 있는 html element tag의 상위 tag들을 한단계씩 올라가며 순차 조회하면서 원하는 Data를 찾는다.
   2) 점 3개 버튼을 클릭한 영상 썸네일의 리스트(**list-ref**) 부분까지만 조회한다. 그렇기 때문에, 점 3개 버튼을 클릭한 영상 썸네일의 리스트 html element tag명(**list-ref**)이 무엇인지 먼저 아는것이 중요하다.

      * **채널 주소** or **channel-id** :
        * 클릭한 점 3개 버튼의 closest에 <span style="font-weight: bold;color:#000;background-color:#FFF;">&lt;ytd-watch-metadata&gt;</span> tag가 있고, &lt;ytd-watch-metadata&gt; tag에 attribute된 "video-id"라는 key의 값이 있고, location.pathname이 "/watch"이고, location.search에 "v=" QUERY key 가 있고, "v" key의 값이 &lt;ytd-watch-metadata&gt; tag가 가진 "video-id" key의 값과 같다면 : 
          * &lt;ytd-watch-metadata&gt; tag 하위에 href가 "/@"로 시작하는 &lt;A&gt; tag가 있다면, href의 시작 문자열인 "/@" 를 제거한 후 나머지 문자열을 decodeURIComponent 한 결과 문자열이 **채널 주소** 가 된다.
          * &lt;ytd-watch-metadata&gt; tag 하위에 href가 "/channel/"로 시작하는 &lt;A&gt; tag가 있다면, href의 시작 문자열인 "/channel/" 를 제거한 후 나머지 문자열을 decodeURIComponent 한 결과 문자열이 **channel-id** 가 된다.
          * 이 &lt;A&gt; tag는 `<a>채널 주소</a>` 이나 `<a>채널 주소<tag>some text</tag></a>` 처럼 string text로 시작하는 형식이어야 한다.
            * &lt;a&gt;&lt;tag&gt;&lt;/tag&gt;&lt;/a&gt; 처럼 어떤 html tag로 시작하면 이건 채널 주소, channel-id 조회용 &lt;A&gt; tag가 아닌거다.
        * 위의 경우가 아니라면,
          * 클릭한 점 3개 버튼의 상위 tag로 한 단계씩 이동할 때마다, 현재 depth의 tag가 포함하고 있는 &lt;A&gt; element tag 중 href가 "/@"로 시작하거나, "/channel/"로 시작하는 첫번째 &lt;A&gt; tag를 찾는다.
          * 이 &lt;A&gt; tag는 `<a>채널 주소</a>` 이나 `<a>채널 주소<tag>some text</tag></a>` 처럼 string text로 시작하는 형식이어야 한다.
            * &lt;a&gt;&lt;tag&gt;&lt;/tag&gt;&lt;/a&gt; 처럼 어떤 html tag로 시작하면 이건 채널 주소, channel-id 조회용 &lt;A&gt; tag가 아닌거다.
          * 찾은 &lt;A&gt; tag가 "/@"로 시작하는 href 를 가졌다면, 시작 문자열인 "/@" 를 제거한 후 나머지 문자열을 decodeURIComponent 한 결과 문자열이 **채널 주소** 가 된다.
          * 찾은 &lt;A&gt; tag가 "/channel/"로 시작하는 href 를 가졌다면, 시작 문자열인 "/channel/" 를 제거한 문자열이 **channel-id** 가 된다.
      * **video-id** :
        * 클릭한 점 3개 버튼의 closest에 &lt;ytd-watch-metadata&gt; tag가 있고, &lt;ytd-watch-metadata&gt; tag에 attribute된 "video-id"라는 key의 값이 있고, location.pathname이 "/watch"이고, location.search에 "v=" QUERY key 가 있고, "v" key의 값이 &lt;ytd-watch-metadata&gt; tag가 가진 "video-id" key의 값과 같다면, "v" QUERY key의 값이 해당 영상의 **video-id**가 된다.
        * 위의 경우가 아니라면,
          * 클릭한 점 3개 버튼의 상위 tag로 한 단계씩 이동할 때마다, 현재 depth의 tag가 포함하고 있는 &lt;A&gt; element tag 중 href가 "/watch"로 시작하고 "v=" QUERY를 가진 첫번째 &lt;A&gt; tag를 찾는다.
          * 찾은 &lt;A&gt; tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 **video-id**가 된다.
      * **채널명** :
        * location.pathname이 "/watch" 이고, 클릭한 점 3개 버튼의 closest에 &lt;ytd-watch-metadata&gt; tag가 없고, 클릭한 점 3개 버튼의 closest에 <span style="font-weight: bold;color:#000;background-color:#FFF;">&lt;yt-lockup-metadata-view-model&gt;</span> tag가 있다면 : 
          * &lt;yt-lockup-metadata-view-model&gt; tag 하위에 `aria-label` 을 가진 tag를 찾음
          * &lt;yt-lockup-metadata-view-model&gt; tag 하위에서 `aria-label`의 값 문자열이 포함된 tag의 textContent를 찾음
            * 예를들어, &lt;yt-lockup-metadata-view-model&gt; tag 하위에 `<div aria-label="aaa bbb ccc 채널로 이동">` 처럼 값에 "aaa bbb ccc"를 포함하고 있는 `aria-label` 을 가진 tag가 있고, 같은 &lt;yt-lockup-metadata-view-model&gt; tag 하위에 `<span>aaa bbb ccc</span>` 처럼 textContent가 "aaa bbb ccc" 인 tag가 있다면, "aaa bbb ccc" 가 **채널명**임
              * "aaa bbb ccc"는 설명을 위한 **예시 채널명**임
              * 내가 찾는 채널명 text 바로 다음에 다른 text를 가진 다른 tag가 있을 수 있으니, 이 부분을 염두해서 코드를 작성해야 한다.
                * ex. &lt;span&gt;aaa bbb ccc &lt;p&gt;viawen &lt;span&gt;tawease &lt;svg /&gt;&lt;/span&gt;&lt;/p&gt;&lt;/span&gt;
              * "채널로 이동" 은 내가 한국이라서 한국어로 나오고, 다른나라는 다른나라 언어로 나올것이기 때문에 "aaa bbb ccc" 만 고려해서 설계함
        * 위의 경우가 아니라면,
          * 클릭한 점 3개 버튼의 상위 tag로 한 단계씩 이동할 때마다, 현재 depth의 tag가 포함하고 있는 &lt;A&gt; element tag 중 href가 "/@"로 시작하거나, "/channel/"로 시작하는 첫번째 &lt;A&gt; tag를 찾는다.
          * 이 &lt;A&gt; tag는 `<a>aaa bbb ccc</a>` 이나 `<a>aaa bbb ccc<tag>some text</tag></a>` 처럼 string text로 시작하는 형식이어야 한다.
            * "aaa bbb ccc" 는 설명을 위한 **예시 채널명** 이다.
            * &lt;a&gt;&lt;tag&gt;&lt;/tag&gt;&lt;/a&gt; 처럼 어떤 html tag로 시작하면 이건 채널명 조회용 &lt;A&gt; tag가 아닌거다.
          * href가 "/@"로 시작하거나, "/channel/"로 시작하는 &lt;A&gt; tag의 textContent가 **채널명**이 된다.
            * 내가 찾는 채널명 text 바로 다음에 다른 text를 가진 다른 tag가 있을 수 있으니, 이 부분을 염두해서 코드를 작성해야 한다.
              * ex. &lt;a&gt;aaa bbb ccc &lt;p&gt;viawen &lt;span&gt;tawease &lt;svg /&gt;&lt;/span&gt;&lt;/p&gt;&lt;/a&gt;
