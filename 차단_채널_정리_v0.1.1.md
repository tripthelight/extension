# 차단 채널 경로 정리 및 필요 Data 배정

## 메인화면
메인화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-browse > ytd-two-column-browse-results-renderer > ytd-rich-grid-renderer`

### long
1. 메인화면에서 long 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 long 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id, channel-id \] 가 필요함
2. long tag flow : `메인화면 tag flow > ytd-rich-item-renderer`
   * long의 채널명 : <br>
     * `long tag flow > a` 중 href 문자열이 "/@" 로 시작하거나 "/channel/"로 시작하는 a의 textContent가 **채널명**이 됨
     * 내가 찾는 text 바로 다음에 다른 text를 가진 다른 tag가 있을 수 있음
       * ex. &lt;a&gt;채널명 &lt;tag&gt;viawen &lt;tag&gt;tawease &lt;/tag&gt; &lt;/tag&gt; &lt;/a&gt;
   * long의 채널 주소 : <br>
     * 이 경우, "@aaa" 형식의 **채널 주소**와, "/channel/aaa" 형식의 **channel-id** 둘 중에 한 개만 있음
       * "aaa" 는 설명을 위한 예시 text
     * `long tag flow > a` 중 href 문자열이 "/@" 로 시작하는 a의 href 문자열에서 시작문자열인 "/@" 를 제거한 후 나머지 문자열을 decodeURIComponent 한 결과 문자열이 **채널 주소**가 됨
     * `long tag flow > a` 중 href 문자열이 "/channel/" 로 시작하는 a의 href 문자열에서 시작문자열인 "/channel/" 제거한 후 나머지 문자열을 decodeURIComponent 한 결과 문자열이 **channel-id** 가 됨
   * long 썸네일 클릭 시 이동하는 url(video-id) : <br>
     * `long tag flow > a` 중 href가 "/watch" 로 시작하고, "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 long 영상의 **video-id**가 됨

### Shorts
1. 메인화면에서 Shorts 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 Shorts 영상의 \[ code, label, pageType, video-id \] 가 필요함
2. Shorts tag flow : `메인화면 tag flow > ytd-rich-section-renderer > ytd-rich-shelf-renderer > ytd-rich-item-renderer`
   * Shorts 썸네일 클릭 시 이동하는 url(video-id) : <br>
     * `Shorts tag flow > a` tag 중 href가 "/shorts/" 로 시작하는 첫번째 a tag 의 href 문자열에서, 시작 문자열인 "/shorts/" 문자열를 제거한 후, 나머지 문자열이 해당 Shorts 영상의 **video-id**가 됨

## Shorts 화면
Shorts 화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-shorts`

1. Shorts 화면에서 해당 Shorts 영상 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 Shorts 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
2. Shorts page tag flow : `Shorts 화면 tag flow > ytd-reel-video-renderer > ytd-player`
   * 해당 Shorts 채널명 : <br>
      * `Shorts page tag flow > yt-reel-channel-bar-view-model > .yt-core-attributed-string > a` 의 href 문자열은 "/@%EC%9B%90%EC%98%81%EC%9D%B4%EC%97%90_%EB%AF%B8%EC%B9%9C_bird/shorts" 형식으로 되어있음
      * "/@%EC%9B%90%EC%98%81%EC%9D%B4%EC%97%90_%EB%AF%B8%EC%B9%9C_bird/shorts" 는 설명을 위한 예시 문자열임.
      * href에서 시작 문자인 "/@"을 삭제하고, 끝 문자열인 "/shorts"를 삭제하고 남은 문자열을 decodeURIComponent 한 결과 문자열이 채널명이 됨
   * 해당 Shorts 채널 주소 : <br>
      * **해당 Shorts 채널명** 과 해당 Shorts 채널 주소는 동일함
   * 해당 Shorts url(video-id) : <br>
      * `Shorts page tag flow > .ytp-chrome-top > .ytp-title > .ytp-title-text > a.ytp-title-link` 의 href문자열은 "https://www.youtube.com/shorts/Pl7lB8X9scg" 형식으로 되어있음. 
      * "Pl7lB8X9scg" 는 설명을 위한 예시 문자열임. 
      * location.pathname 의 "/shorts/Pl7lB8X9scg" 와 href 문자열의 "/shorts/Pl7lB8X9scg" 부분이 같다면 "Pl7lB8X9scg" 문자열이 video-id가 됨



## 채널페이지
**공통**
1. 채널페이지 상단 `body > ytd-app > ytd-page-manager > ytd-browse > ytd-tabbed-page-header > tp-yt-app-header-layout > tp-yt-app-header > yt-page-header-renderer > yt-dynamic-text-view-model > h1` 의 textContent 문자를 decodeURIComponent 한 결과 문자열이 해당 채널의 **채널명**임
2. 채널페이지 location.pathname 는 "/@aaa/featured" 형식으로 되어있음
   - 홈탭 : "/@aaa/featured", 동영상탭 : "/@aaa/videos",  Shorts탭 : "/@aaa/shorts", 라이브탭 : "/@aaa/streams"
   - "@aaa" 는 설명을 위한 예시 채널 주소임
   - 채널페이지 location.pathname 의 시작 문자인 "/@" 를 삭제 후, 채널명 다음에 있는 "/" 로 시작하는 모든 문자를 삭제한 문자를 decodeURIComponent 한 결과 문자열이 해당 채널의 **채널 주소**가 됨

### 홈 탭
- 채널페이지 홈 탭 화면 tag flow : `body > ytd-app > ytd-page-manager > ytd-browse > ytd-two-column-browse-results-renderer > ytd-section-list-renderer`
- \[ 최상단에서 현재 재생 중인 영상, 실시간 스트림 n개, 그 하위 영상 리스트 \] 각각의 section은 유튜브 주인장의 설정에 따라 있을수도 있고, 없을수도 있음

#### 최상단에서 현재 재생 중인 영상
1. 채널페이지 홈 탭 화면 진입 시 상단에서 바로 재생되는 영상인데, 여기 section에는 점 3개 버튼 없음
2. Channel page home live tag flow : `채널페이지 홈 탭 화면 tag flow > ytd-item-section-renderer > ytd-channel-video-player-renderer`

#### 실시간 스트림 n개
1. 채널페이지 홈 탭 화면에서 하위 **실시간 스트림 n개** Section 에 있는 영상 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
   - 채널명, 채널 주소를 알아내는 방법은 위에 **공통** 참고
2. Channel page home stream tag flow : `채널페이지 홈 탭 화면 tag flow > ytd-item-section-renderer > ytd-channel-featured-content-renderer > ytd-video-renderer`
   * 해당영상 url(video-id) : <br>
      `Channel page home stream tag flow > a` 중 href가 "/watch" 로 시작하고 "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 video-id가 됨

#### 그 하위 영상 리스트(동영상위에 점3개 버튼 있는 동영상만 해당)
1. 채널페이지 홈 탭 화면에서 하위 Section 에 있는 영상 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
   - 채널명, 채널 주소를 알아내는 방법은 위에 **공통** 참고
2. Channel page home section tag flow : `채널페이지 홈 탭 화면 tag flow > ytd-item-section-renderer > ytd-shelf-renderer > yt-horizontal-list-renderer > ytd-grid-video-renderer`
   * 해당영상 url(video-id) : <br>
      `Channel page home section tag flow > a` 중 href가 "/watch" 로 시작하고 "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 video-id가 됨 

#### Shorts

### 동영상 탭
1. 채널페이지 동영상 탭 화면에서 영상 썸네일들 중 하나의 element 하위에 있는 점 3개 버튼을 클릭할 경우, 클릭한 해당영상의 \[ code, label, pageType, 채널명, 채널 주소, video-id \] 가 필요함
   - 채널명, 채널 주소를 알아내는 방법은 위에 **공통** 참고
2. Channel page 동영상 탭 tag flow : `body > ytd-app > ytd-page-manager > ytd-browse > ytd-two-column-browse-results-renderer > ytd-rich-grid-renderer > ytd-rich-item-renderer`
   * 해당영상 url(video-id) : <br>
      `Channel page 동영상 탭 tag flow > a` 중 href가 "/watch" 로 시작하고 "v=" QUERY를 가진 첫번째 a tag의 href 문자열에서 "v=" QUERY key의 값이 해당 영상의 video-id가 됨
